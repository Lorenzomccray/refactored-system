# KB MAINTENANCE PROTOCOL

## Purpose
Keep `kb/*` synchronized with canonical docs and runtime truth.

## Update triggers
- canonical meaning changes
- workflow meaning changes
- safety or validation meaning changes
- runtime reality stays different long enough to matter

## Rules
- KB is navigation and maintained guidance, not competing canon.
- If KB and canonical docs differ, canonical docs win.
- If KB is stale, mark and update it.
- Keep KB pages compact and retrieval-friendly.
