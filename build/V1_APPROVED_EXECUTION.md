## Approval process

1. A patch proposal is created in `build/V1_PATCH_PROPOSAL.md`.
2. The planner reviews it against `build/V1_SAFE_PLANNER.md`.
3. If approved, an entry is added to this file before execution begins.
4. After execution, a receipt is filed in `docs/EXECUTION_RECEIPTS.md`.

## Rule

Do not execute any V1 change that does not have a corresponding entry in this log.
