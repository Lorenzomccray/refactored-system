# Canonical precedents

- The repo is an early implementation seed, not the full canonical architecture.
- Typed contracts are the preferred direction over opaque strings.
- Planner, builder, and verifier staging is the preferred execution model.
- Browser capability is not enough; policy and verification must govern it.
- Runtime tool claims must be revalidated instead of assumed.
- Docs are truth, runtime docs are current truth, and chats are working context.
